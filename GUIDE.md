# Руководство для начинающих — Multi-Browser Operator

Это пошаговая инструкция для тех, кто впервые работает с проектом.

---

## Что делает программа

Multi-Browser Operator позволяет управлять несколькими окнами одновременно.
Вы работаете в одном «главном» окне (Master), а программа автоматически повторяет все ваши действия (мышь, клавиатура, прокрутка) во всех выбранных «ведомых» окнах (Slave).

**Типичный сценарий:** 15–20 браузеров, в которых нужно заполнить одинаковые формы или пройти одинаковые шаги.

---

## Требования

| Что нужно | Версия |
|---|---|
| Windows | 10 или 11 |
| Python | 3.10 или новее |
| pip | поставляется с Python |

> Программа **не работает** на macOS и Linux — она использует Win32 API напрямую.

## Установка

### Шаг 1 — Установите Python

Скачайте Python с [python.org](https://www.python.org/downloads/).
При установке поставьте галочку **"Add Python to PATH"**.

Проверьте установку:
```
python --version
```
Должно вывести что-то вроде `Python 3.12.0`.

### Шаг 2 — Скачайте проект

Если у вас установлен Git:
```
git clone <адрес-репозитория>
cd -Multi-Browser-Operator-
```

Или скачайте ZIP-архив с GitHub и распакуйте его.

### Шаг 3 — Установите зависимости

Откройте командную строку в папке проекта и выполните:
```
pip install -r requirements.txt
```

Это установит PyQt5 (интерфейс) и PyInstaller (для сборки .exe).

### Шаг 4 — Запустите программу

```
python run.py
```

Откроется окно с интерфейсом.

## Как пользоваться

### 1. Обновите список окон

Нажмите кнопку **«Обновить список окон»**.
В таблице появятся все видимые окна на вашем рабочем столе.

### 2. Назначьте главное окно (Master)

Кликните на строку нужного окна в таблице (например, первый браузер).
Нажмите **«Назначить выбранное»**.
В блоке «Ведущее окно» появится название выбранного окна.

### 3. Отметьте ведомые окна (Slave)

Поставьте галочки напротив всех окон, которые должны повторять действия.
Кнопки **«Выбрать все»** / **«Снять все»** упрощают выбор.

> Главное окно автоматически убирается из ведомых, даже если вы поставили галочку.

### 4. Запустите синхронизацию

Нажмите **«▶ Старт»**.
Кнопка станет красной («⏹ Стоп»), и внизу появится счётчик событий.

Теперь всё, что вы делаете в главном окне, немедленно повторяется во всех ведомых.

### 5. Пауза и возобновление

- Кнопка **«⏸ Пауза (F8)»** — приостановить синхронизацию.
- Нажмите ещё раз (или **F8**) — продолжить.

> **Важно:** F8 работает только когда окно программы активно. Для паузы из любого места используйте иконку в системном трее.

### 6. Остановка

Нажмите **«⏹ Стоп»** — синхронизация отключится.
Вы можете выбрать другие окна и снова нажать «Старт».

### 7. Системный трей

Закрытие окна не завершает программу — она сворачивается в трей (правый нижний угол).
- **Двойной клик** по иконке — открыть окно.
- **Правый клик** — меню с «Пауза» и «Выход».

## Масштабирование координат

Опция **«Масштабировать координаты»** включает пропорциональный пересчёт координат.

| Режим | Когда использовать |
|---|---|
| Выключено (по умолчанию) | Все окна одинакового размера |
| Включено | Окна разных размеров |

При включённом масштабировании клик в центре главного окна попадёт в центр каждого ведомого, независимо от их реальных размеров.

## Частые проблемы

### Программа не реагирует на действия в браузере

Некоторые браузеры игнорируют `PostMessage`-события (используют низкоуровневый ввод).
Попробуйте:
- Убедитесь, что главное окно активно (вы кликаете именно в нём).
- Попробуйте кликнуть внутри области страницы, не на адресной строке.

### В списке окон нет нужного окна

Нажмите **«Обновить список окон»** — список не обновляется автоматически.
Убедитесь, что окно не свёрнуто (minimized) при обновлении.

### Окна не синхронизируются после обновления списка

После нажатия «Обновить» все настройки сбрасываются — нужно заново назначить мастер и слейвы.
Кнопка «Обновить» недоступна во время синхронизации (это сделано специально).

### Ошибка при запуске `python run.py`

Убедитесь, что вы запускаете команду из папки проекта, а не из другого места:
```
cd путь\до\-Multi-Browser-Operator-
python run.py
```

## Структура проекта (кратко)

```
run.py              — точка входа, запуск приложения
src/
  gui.py            — интерфейс (PyQt5)
  engine.py         — логика синхронизации
  hooks.py          — перехват мыши и клавиатуры (Win32)
  sender.py         — отправка событий в ведомые окна
  window_manager.py — получение списка окон
  winapi.py         — определения Win32 API через ctypes
  config.py         — сохранение настроек в JSON
```
