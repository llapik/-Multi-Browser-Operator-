# Multi-Browser Operator

Десктопное приложение для синхронного управления несколькими окнами на Windows. Все действия мыши и клавиатуры в «ведущем» окне автоматически дублируются во всех «ведомых» окнах.

## Возможности

- **Синхронизация мыши** — движение курсора, клики (ЛКМ/ПКМ/СКМ), прокрутка колесиком
- **Синхронизация клавиатуры** — ввод текста, горячие клавиши, навигация
- **Управление до 20+ окон** одновременно
- **Режим паузы** (F8) — временная остановка синхронизации
- **Масштабирование координат** — пропорциональный пересчёт при разных размерах окон
- **Системный трей** — приложение сворачивается в область уведомлений
- **Универсальность** — работает с любыми оконными приложениями (браузеры, блокнот, и т.д.)

## Системные требования

- Windows 10/11
- Python 3.10+

## Установка

```bash
git clone <repo-url>
cd -Multi-Browser-Operator-
pip install -r requirements.txt
```

## Запуск

```bash
python run.py
```

```bash
build.bat
```

Готовый файл: `dist/MultiBrowserOperator.exe`

## Как пользоваться

1. Запустите приложение
2. Нажмите **«Обновить список окон»** для обновления списка
3. Выберите строку в таблице и нажмите **«Назначить выбранное»** — это будет ведущее окно (Master)
4. Отметьте галочками ведомые окна (Slaves)
5. Нажмите **«Старт»** — синхронизация начата
6. Нажмите **F8** для паузы / продолжения
7. Закройте окно — приложение свернётся в трей

## Архитектура

```
src/
├── winapi.py          — определения Win32 API (ctypes)
├── window_manager.py  — перечисление и управление окнами
├── hooks.py           — низкоуровневые хуки мыши и клавиатуры
├── sender.py          — отправка событий в ведомые окна (PostMessage)
├── engine.py          — движок синхронизации
├── gui.py             — интерфейс PyQt5 с системным треем
└── config.py          — сохранение настроек (JSON)
```

### Принцип работы

1. Глобальные хуки (`WH_MOUSE_LL`, `WH_KEYBOARD_LL`) перехватывают ввод
2. Движок проверяет, что фокус на ведущем окне
3. Экранные координаты преобразуются в локальные координаты окна
4. `PostMessageW` отправляет события в каждое ведомое окно без перехвата фокуса
